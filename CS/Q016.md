# [Q016] OS 가상 메모리 동작 원리

**Q.** 운영체제(OS)의 가상 메모리(Virtual Memory)는 물리 메모리의 한계를 극복하고 여러 프로세스가 독립적으로 실행될 수 있도록 돕는 핵심 기술입니다. 가상 메모리의 기본적인 동작 원리(페이징, 스와핑)를 설명하고, 가상 메모리가 제공하는 주요 이점(예: 메모리 보호, 다중 프로그래밍)에 대해 설명해주세요. 또한, 페이지 교체 알고리즘(예: LRU, FIFO)이 가상 메모리 성능에 미치는 영향에 대해서도 간략히 설명해주세요.

## 가상 메모리 동작 원리 및 이점, 성능 영향

### 1. 가상 메모리 동작 원리 (페이징, 스와핑)

가상 메모리는 실제 물리 메모리(RAM)보다 훨씬 큰 메모리 공간을 프로세스에 제공하는 기술입니다. 이는 **페이징(Paging)**과 **스와핑(Swapping)**이라는 두 가지 핵심 메커니즘을 통해 구현됩니다.

*   **페이징 (Paging):**
    *   프로세스에 할당되는 가상 주소 공간을 고정된 크기의 블록인 **페이지(Page)**로 나눕니다.
    *   물리 메모리 또한 페이지와 동일한 크기의 블록인 **프레임(Frame)**으로 나눕니다.
    *   운영체제는 **페이지 테이블(Page Table)**을 사용하여 각 가상 페이지가 물리 메모리의 어느 프레임에 매핑되어 있는지 기록합니다.
    *   프로세스가 가상 주소에 접근하면, MMU(Memory Management Unit)가 페이지 테이블을 참조하여 해당 가상 주소를 물리 주소로 변환합니다.
    *   페이지 테이블 엔트리에는 해당 페이지가 물리 메모리에 적재되어 있는지(유효 비트) 등의 정보가 포함됩니다.

*   **스와핑 (Swapping):**
    *   프로세스가 필요로 하는 모든 페이지가 물리 메모리에 동시에 적재될 수 없을 때 사용됩니다.
    *   물리 메모리에 공간이 부족하면, 운영체제는 현재 사용되지 않거나 우선순위가 낮은 물리 메모리의 페이지를 하드디스크의 **스와프 공간(Swap Space)**으로 옮깁니다. 이를 **페이지 아웃(Page Out)** 또는 **스와프 아웃(Swap Out)**이라고 합니다.
    *   이후 해당 페이지가 다시 필요해지면, 스와프 공간에서 물리 메모리로 다시 불러옵니다. 이를 **페이지 인(Page In)** 또는 **스와프 인(Swap In)**이라고 합니다.
    *   스와핑은 물리 메모리의 한계를 극복하지만, 디스크 I/O는 RAM 접근보다 훨씬 느리므로 과도한 스와핑은 성능 저하(쓰레싱)를 유발할 수 있습니다.

### 2. 가상 메모리의 주요 이점

가상 메모리는 다음과 같은 중요한 이점들을 제공합니다.

*   **물리 메모리 한계 극복:** 프로세스는 실제 물리 메모리 크기에 구애받지 않고 더 큰 가상 주소 공간을 사용할 수 있습니다. 이는 대용량 데이터를 처리하는 애플리케이션에 필수적입니다.
*   **다중 프로그래밍(Multiprogramming) 효율 증대:** 여러 프로세스가 동시에 실행될 수 있도록 돕습니다. 각 프로세스는 독립적인 가상 주소 공간을 가지므로, 물리 메모리의 연속적인 공간을 필요로 하지 않아 메모리 파편화 문제를 줄이고 더 많은 프로세스를 동시에 적재할 수 있습니다.
*   **메모리 보호(Memory Protection):** 각 프로세스는 자신만의 독립적인 가상 주소 공간을 가지므로, 한 프로세스가 다른 프로세스의 메모리 영역을 침범하거나 운영체제의 커널 영역에 접근하는 것을 방지합니다. 이는 시스템 안정성과 보안을 강화합니다.
*   **프로그램 공유 용이:** 여러 프로세스가 동일한 라이브러리나 코드 영역을 공유할 때, 해당 페이지를 물리 메모리에 한 번만 적재하고 여러 프로세스의 페이지 테이블에서 해당 물리 프레임을 가리키도록 하여 메모리 사용 효율을 높일 수 있습니다.
*   **프로그램 로딩 및 실행 효율성:** 프로그램의 전체 코드를 물리 메모리에 한 번에 로드할 필요 없이, 필요한 부분(페이지)만 그때그때 로드하여 실행할 수 있습니다(요구 페이징). 이는 프로그램 시작 시간을 단축하고 메모리 사용을 최적화합니다.

### 3. 페이지 교체 알고리즘이 가상 메모리 성능에 미치는 영향

페이지 교체 알고리즘은 물리 메모리가 가득 찼을 때, 새로 필요한 페이지를 적재하기 위해 어떤 페이지를 스와프 아웃할지 결정하는 전략입니다. 이 알고리즘의 선택은 가상 메모리 시스템의 성능에 지대한 영향을 미칩니다.

*   **목표:** 페이지 교체 알고리즘의 주된 목표는 **페이지 부재(Page Fault)** 발생률을 최소화하는 것입니다. 페이지 부재는 필요한 페이지가 물리 메모리에 없어 디스크에서 불러와야 하는 상황으로, 매우 느린 디스크 I/O를 수반하므로 시스템 성능을 크게 저하시킵니다.

*   **주요 알고리즘:**
    *   **FIFO (First-In, First-Out):** 가장 먼저 물리 메모리에 들어온 페이지를 먼저 내보냅니다. 구현이 간단하지만, 오랫동안 사용되지 않았더라도 중요한 페이지가 교체될 수 있어 비효율적일 수 있습니다.
    *   **LRU (Least Recently Used):** 가장 오랫동안 사용되지 않은 페이지를 교체합니다. 미래에도 사용될 가능성이 낮다고 가정하여 효율적인 교체를 기대할 수 있습니다. 하지만 각 페이지의 사용 시점을 추적해야 하므로 구현 비용이 높습니다.

*   **성능 영향:**
    *   **페이지 부재율:** 효율적인 페이지 교체 알고리즘은 페이지 부재율을 낮춰 디스크 I/O 횟수를 줄이고, 결과적으로 시스템의 전반적인 응답 속도와 처리량을 향상시킵니다.
    *   **쓰레싱(Thrashing):** 비효율적인 알고리즘이나 과도한 다중 프로그래밍으로 인해 페이지 부재가 너무 자주 발생하여 CPU가 대부분의 시간을 페이지 스와핑에만 소비하고 실제 작업을 거의 하지 못하는 상태를 쓰레싱이라고 합니다. 이는 시스템 성능을 극도로 저하시키며, 페이지 교체 알고리즘의 선택이 쓰레싱 발생 여부에 큰 영향을 미칩니다.
    *   **오버헤드:** 알고리즘 자체의 복잡도(예: LRU의 사용 시간 추적)는 추가적인 CPU 및 메모리 오버헤드를 발생시킬 수 있습니다. 따라서 성능 향상과 오버헤드 사이의 균형을 고려해야 합니다.

결론적으로, 가상 메모리는 물리 메모리의 한계를 극복하고 다중 프로그래밍 환경에서 메모리 보호와 효율성을 제공하는 필수적인 기술이며, 페이지 교체 알고리즘은 이러한 가상 메모리 시스템의 성능을 좌우하는 핵심 요소입니다.
