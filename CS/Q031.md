## 컨텍스트 스위칭(Context Switching)

### 1. 핵심 정의

컨텍스트 스위칭은 CPU가 현재 실행 중인 프로세스(또는 스레드)의 상태를 저장하고, 다른 프로세스(또는 스레드)의 상태를 로드하여 실행을 재개하는 과정입니다. 이는 단일 CPU가 여러 프로세스를 동시에 실행하는 것처럼 보이게 하는 시분할 멀티태스킹의 핵심 메커니즘입니다.

### 2. 동작 원리 및 시나리오

컨텍스트 스위칭은 다음과 같은 시나리오에서 발생합니다.

*   **타이머 인터럽트 (시분할 시스템):** 운영체제가 각 프로세스에 할당된 CPU 시간(타임 슬라이스)이 만료되면 타이머 인터럽트를 발생시켜 현재 프로세스를 중단하고 다른 프로세스로 전환합니다.
*   **I/O 요청 또는 시스템 호출:** 프로세스가 I/O 작업(예: 디스크 읽기, 네트워크 통신)을 요청하거나 특정 시스템 호출을 수행하여 대기 상태로 전환될 때, CPU는 다른 실행 가능한 프로세스로 전환됩니다.
*   **우선순위가 높은 프로세스의 등장:** 우선순위가 더 높은 프로세스가 준비 큐에 들어오면, 현재 실행 중인 프로세스를 선점하고 새로운 프로세스로 전환될 수 있습니다.
*   **세마포어, 뮤텍스 등 동기화 기본 요소에 의한 대기:** 프로세스가 공유 자원에 접근하기 위해 락을 획득하려다 실패하여 대기 상태로 전환될 때 발생합니다.

컨텍스트 스위칭 과정에서 저장되고 복원되는 정보는 다음과 같습니다.

*   **CPU 레지스터:** 프로그램 카운터(PC), 스택 포인터(SP), 범용 레지스터 등 CPU 내부의 모든 레지스터 값.
*   **프로세스 상태:** 프로세스의 현재 상태(실행, 대기, 준비 등).
*   **메모리 관리 정보:** 페이지 테이블 또는 세그먼트 테이블 정보 (MMU 관련).
*   **운영체제 자원 정보:** 열린 파일 목록, 할당된 I/O 장치 등.

이러한 정보들은 일반적으로 해당 프로세스의 **프로세스 제어 블록(PCB)**에 저장됩니다.

### 3. 시스템 성능에 미치는 영향 및 최소화 노력

컨텍스트 스위칭은 필연적으로 **오버헤드**를 발생시켜 시스템 성능에 영향을 미칩니다.

*   **시간 오버헤드:** CPU 레지스터 및 메모리 정보 저장/복원, PCB 업데이트, 스케줄러 실행 등에 소요되는 시간. 이 시간 동안 CPU는 실제 유용한 작업을 수행하지 못합니다.
*   **캐시 오버헤드:** 새로운 프로세스로 전환될 때 기존 프로세스의 캐시 데이터가 무효화되거나 교체되어, 새로운 프로세스가 데이터를 다시 캐시에 로드해야 하므로 캐시 미스율이 증가하고 성능 저하를 유발할 수 있습니다.

운영체제는 이러한 오버헤드를 최소화하기 위해 다음과 같은 노력을 합니다.

*   **스레드 사용:** 프로세스 내의 스레드 간 컨텍스트 스위칭은 프로세스 간 스위칭보다 오버헤드가 적습니다. 스레드는 동일한 주소 공간을 공유하므로 메모리 관리 정보(페이지 테이블 등)를 변경할 필요가 없기 때문입니다.
*   **최적화된 스케줄링 알고리즘:** 컨텍스트 스위칭 발생 빈도를 줄이면서도 공정성과 응답성을 유지하는 스케줄링 알고리즘을 사용합니다. 예를 들어, 타임 슬라이스를 너무 짧게 설정하면 스위칭이 잦아져 오버헤드가 커지고, 너무 길게 설정하면 응답성이 저하될 수 있습니다.
*   **하드웨어 지원:** 일부 CPU는 컨텍스트 스위칭에 필요한 레지스터 저장/복원 작업을 하드웨어적으로 지원하여 속도를 향상시킵니다.
*   **경량 프로세스(LWP) 또는 파이버(Fiber) 사용:** 사용자 수준에서 컨텍스트 스위칭을 관리하여 커널 오버헤드를 줄이는 방식을 사용하기도 합니다.
