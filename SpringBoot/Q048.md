# [Q048] Spring Data JPA Auditing 활용

> 🌱 Spring Boot · 2026-07-05

**Q.** Spring Data JPA의 Auditing 기능은 엔티티의 생성 및 수정 시점, 그리고 생성/수정자를 자동으로 기록하는 데 유용합니다. Auditing 기능을 설정하고 활용하는 방법을 설명하고, 이 기능이 애플리케이션의 데이터 무결성 및 추적 가능성을 어떻게 향상시키는지 설명해주세요. 또한, Auditing 사용 시 고려해야 할 사항과 커스터마이징 방안에 대해 설명해주세요.

## 🧑‍💻 나의 답변

## Spring Data JPA Auditing

### 1. 핵심 정의

Spring Data JPA Auditing은 엔티티의 생성 및 수정 시점(createdAt, updatedAt)과 생성/수정자(createdBy, updatedBy) 정보를 자동으로 기록해주는 기능입니다. 이를 통해 반복적인 코드 작성을 줄이고, 데이터의 이력 관리를 용이하게 합니다.

### 2. 동작 원리 및 설정/활용 방법

1.  **설정 활성화**: `@EnableJpaAuditing` 어노테이션을 Spring Boot 애플리케이션의 메인 클래스 또는 `@Configuration` 클래스에 추가하여 Auditing 기능을 활성화합니다.
2.  **AuditorAware 구현**: `AuditorAware<T>` 인터페이스를 구현하여 현재 로그인한 사용자(또는 시스템)의 정보를 제공하는 빈을 등록합니다. `getCurrentAuditor()` 메서드가 현재 감사자 정보를 반환합니다.
3.  **엔티티 적용**:
    *   `@EntityListeners(AuditingEntityListener.class)` 어노테이션을 엔티티 클래스에 추가합니다.
    *   `@CreatedDate`, `@LastModifiedDate` 어노테이션을 각각 생성일, 수정일 필드에 적용합니다.
    *   `@CreatedBy`, `@LastModifiedBy` 어노테이션을 각각 생성자, 수정자 필드에 적용합니다.
    *   일반적으로 `BaseEntity`와 같은 추상 클래스를 만들어 공통 Auditing 필드를 정의하고, 다른 엔티티들이 이를 상속받아 사용합니다.

### 3. 데이터 무결성 및 추적 가능성 향상

*   **데이터 무결성**: 생성 및 수정 시점과 주체를 자동으로 기록함으로써, 데이터가 언제 누구에 의해 변경되었는지에 대한 신뢰할 수 있는 정보를 제공하여 데이터의 신뢰성을 높입니다.
*   **추적 가능성**: 특정 데이터의 변경 이력을 쉽게 추적할 수 있게 하여 문제 발생 시 원인 분석 및 책임 소재 파악을 용이하게 합니다. 이는 감사(Audit) 요구사항을 충족하는 데 필수적입니다.
*   **개발 생산성**: 반복적인 감사 필드 관리 로직을 제거하여 개발자가 핵심 비즈니스 로직에 집중할 수 있도록 돕습니다.

### 4. Auditing 사용 시 고려사항 및 커스터마이징 방안

*   **AuditorAware 구현**: `AuditorAware` 구현 시, 보안 컨텍스트에서 사용자 정보를 안전하게 가져오는 방법을 고려해야 합니다. (예: Spring Security의 `SecurityContextHolder`)
*   **필드 타입**: 날짜/시간 필드는 `LocalDateTime` 또는 `Instant`를 사용하는 것이 일반적이며, 생성/수정자 필드는 `String` 또는 사용자 ID 타입으로 정의할 수 있습니다.
*   **트랜잭션 경계**: Auditing은 JPA 엔티티 생명주기 이벤트에 의존하므로, 트랜잭션 내부에서 엔티티가 저장되거나 업데이트될 때만 동작합니다.
*   **커스터마이징**:
    *   **날짜/시간 포맷**: `@CreatedDate`, `@LastModifiedDate` 어노테이션에 `Temporal` 어노테이션을 함께 사용하여 저장될 날짜/시간의 정밀도를 조절할 수 있습니다.
    *   **기본값 변경**: `AuditingEntityListener`는 기본적으로 `null` 값을 허용하지만, 필요에 따라 엔티티 필드에 `nullable = false` 제약 조건을 추가할 수 있습니다.
    *   **이벤트 리스너 확장**: `AuditingEntityListener` 외에 사용자 정의 JPA 엔티티 리스너를 구현하여 Auditing 로직을 더욱 세밀하게 제어하거나 추가적인 로직을 삽입할 수 있습니다.
