## 데이터베이스 정규화(Normalization) 및 비정규화(Denormalization) 설명

### 1. 데이터베이스 정규화(Normalization)

**목적:** 데이터베이스 내의 데이터 중복을 최소화하고, 데이터 무결성을 유지하며, 이상 현상(Anomaly) 발생을 방지하여 데이터베이스를 효율적으로 관리하기 위함입니다.

**정규형 개념:**

*   **1차 정규형 (1NF):**
    *   모든 속성(컬럼)은 원자값(Atomic Value)을 가져야 합니다. 즉, 하나의 셀에는 하나의 값만 존재해야 하며, 반복되는 그룹이 없어야 합니다.
    *   모든 레코드(행)는 유일하게 식별 가능해야 합니다. (기본 키 설정 가능)
    *   **예시:** 한 컬럼에 여러 개의 전화번호가 아닌, 각 전화번호를 별도의 컬럼이나 레코드로 분리.

*   **2차 정규형 (2NF):**
    *   1차 정규형을 만족해야 합니다.
    *   모든 비기본 키 속성(Non-key Attribute)은 기본 키 전체에 완전 함수 종속(Full Functional Dependency)이어야 합니다. 즉, 복합 키(Composite Key)의 일부에만 종속되는 속성이 없어야 합니다.
    *   **예시:** (주문번호, 상품번호)가 기본 키일 때, 상품명은 상품번호에만 종속되므로 별도의 테이블로 분리.

*   **3차 정규형 (3NF):**
    *   2차 정규형을 만족해야 합니다.
    *   기본 키가 아닌 모든 속성은 기본 키에 직접적으로 종속되어야 하며, 이행적 함수 종속(Transitive Functional Dependency)이 없어야 합니다. 즉, 비기본 키 속성이 다른 비기본 키 속성에 종속되는 경우가 없어야 합니다.
    *   **예시:** (주문번호)가 기본 키일 때, '고객ID'가 '고객명'을 결정하고 '고객명'이 '고객등급'을 결정하는 경우, '고객등급'은 '고객ID'에 이행적으로 종속되므로 '고객' 테이블로 분리.

**정규화의 장점:**

*   **데이터 중복 최소화:** 저장 공간을 절약하고 일관성을 유지합니다.
*   **데이터 무결성 유지:** 데이터의 정확성과 신뢰성을 높입니다.
*   **이상 현상(삽입, 갱신, 삭제 이상) 방지:** 데이터 변경 시 발생할 수 있는 오류를 줄입니다.
*   **데이터베이스 구조의 유연성 및 확장성 향상:** 변경에 용이하고 새로운 요구사항을 반영하기 쉽습니다.
*   **쿼리 작성 용이성:** 명확한 데이터 구조로 인해 쿼리 작성이 직관적입니다.

### 2. 비정규화(Denormalization)

**고려하는 상황:**

정규화는 데이터 무결성과 일관성을 높이지만, 여러 테이블 간의 조인(JOIN) 연산이 많아져 쿼리 성능이 저하될 수 있습니다. 특히, 다음과 같은 상황에서 비정규화를 고려할 수 있습니다.

*   **읽기(Read) 작업이 압도적으로 많은 시스템:** OLAP(Online Analytical Processing) 시스템이나 보고서 생성 등 조회 성능이 매우 중요한 경우.
*   **복잡한 조인 연산으로 인한 성능 저하가 심각할 때:** 여러 테이블을 조인해야만 얻을 수 있는 데이터를 미리 결합하여 저장.
*   **특정 쿼리의 응답 시간이 매우 중요할 때:** 실시간 서비스 등 빠른 응답이 필수적인 경우.
*   **데이터 중복이 허용 가능한 수준이며, 성능 향상 이점이 더 클 때.**

**얻을 수 있는 이점:**

*   **쿼리 성능 향상:** 조인 연산 감소로 데이터 조회 속도가 빨라집니다.
*   **개발 편의성 증대:** 복잡한 조인 쿼리 대신 단순한 쿼리로 데이터를 가져올 수 있습니다.
*   **데이터 접근 단순화:** 필요한 데이터가 한 테이블에 모여 있어 접근이 용이합니다.

**발생할 수 있는 문제점:**

*   **데이터 중복 증가:** 저장 공간이 낭비되고, 데이터 불일치(Inconsistency) 발생 가능성이 높아집니다.
*   **데이터 무결성 저해:** 중복된 데이터 중 일부만 수정될 경우 데이터가 일치하지 않는 문제가 발생할 수 있습니다.
*   **이상 현상 발생 가능성 증가:** 삽입, 갱신, 삭제 시 더 많은 데이터를 수정해야 하거나, 데이터 불일치로 인한 오류가 발생할 수 있습니다.
*   **데이터 모델의 복잡성 증가:** 중복된 데이터를 관리하기 위한 추가적인 로직이 필요할 수 있습니다.
*   **유연성 저하:** 데이터 구조 변경 시 더 많은 영향을 미칠 수 있습니다.

**결론:**

정규화는 데이터베이스 설계의 기본 원칙이지만, 실제 운영 환경에서는 성능 최적화를 위해 비정규화를 전략적으로 고려할 필요가 있습니다. 비정규화는 신중하게 접근해야 하며, 성능 향상 이점과 데이터 무결성 저해 위험 사이의 균형점을 찾는 것이 중요합니다.
