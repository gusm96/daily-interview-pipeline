## Java 8 Stream API: 중간 연산, 최종 연산, 지연 평가

### 1. 중간 연산(Intermediate Operations)과 최종 연산(Terminal Operations)의 차이점

Stream API는 컬렉션 데이터를 파이프라인 형태로 처리하며, 이 파이프라인은 **중간 연산**과 **최종 연산**으로 구성됩니다.

*   **중간 연산 (Intermediate Operations):**
  *   **특징:** 스트림을 반환하며, 다른 중간 연산이나 최종 연산을 연결할 수 있습니다. 실제 데이터 처리는 수행하지 않고, 스트림 파이프라인을 구성하는 역할만 합니다.
  *   **지연 평가:** 중간 연산은 즉시 실행되지 않고, 최종 연산이 호출될 때까지 기다립니다.
  *   **예시:** `filter()`, `map()`, `sorted()`, `distinct()`, `limit()`, `skip()` 등

*   **최종 연산 (Terminal Operations):**
  *   **특징:** 스트림을 소비하여 결과를 반환합니다 (컬렉션, 단일 값, `void` 등). 최종 연산이 호출되어야 비로소 중간 연산들이 실행되고 실제 데이터 처리가 이루어집니다.
  *   **즉시 실행:** 최종 연산은 호출되는 즉시 실행됩니다.
  *   **스트림 소모:** 한 번 최종 연산이 실행된 스트림은 다시 사용할 수 없습니다.
  *   **예시:** `forEach()`, `collect()`, `reduce()`, `count()`, `min()`, `max()`, `sum()`, `average()`, `anyMatch()`, `allMatch()`, `noneMatch()`, `findFirst()`, `findAny()` 등

**요약:** 중간 연산은 "무엇을 할지"를 정의하고, 최종 연산은 "언제 할지"를 결정하며 실제 작업을 시작합니다.

### 2. 지연 평가(Lazy Evaluation)가 Stream API 성능에 미치는 영향

**지연 평가**는 Stream API의 핵심적인 성능 최적화 메커니즘입니다. 중간 연산은 최종 연산이 호출되기 전까지는 실제 데이터 처리를 수행하지 않고, 단지 "어떤 작업을 할 것인지"에 대한 정보만 저장합니다.

**성능 영향:**

1.  **불필요한 연산 방지:** 최종 연산이 필요한 만큼만 데이터를 처리하므로, 전체 데이터에 대해 모든 중간 연산을 수행할 필요가 없습니다. 이는 특히 대규모 데이터셋에서 큰 성능 이점을 제공합니다.
2.  **무한 스트림 처리 가능:** 지연 평가 덕분에 `Stream.generate()`와 같은 무한 스트림도 효율적으로 처리할 수 있습니다. `limit()`와 같은 중간 연산을 통해 필요한 만큼만 요소를 생성하고 처리할 수 있기 때문입니다.
3.  **최적화된 실행 계획:** JVM은 최종 연산이 호출될 때 전체 스트림 파이프라인을 분석하여 가장 효율적인 실행 계획을 수립합니다. 예를 들어, `filter()`와 `map()` 연산이 연속될 경우, 각 요소를 한 번만 순회하면서 두 연산을 동시에 적용하는 방식으로 최적화할 수 있습니다.

**예시 코드:**

다음 예시를 통해 지연 평가의 효과를 살펴보겠습니다.

```java
import java.util.Arrays;
import java.util.List;
import java.util.stream.Collectors;

public class StreamLazyEvaluation {

  public static void main(String[] args) {
      List<String> names = Arrays.asList("Alice", "Bob", "Charlie", "David", "Eve");

      System.out.println("--- 지연 평가 예시 시작 ---");

      List<String> filteredAndMappedNames = names.stream()
          .filter(name -> {
              System.out.println("필터링 중: " + name); // 중간 연산
              return name.startsWith("A");
          })
          .map(name -> {
              System.out.println("매핑 중: " + name); // 중간 연산
              return name.toUpperCase();
          })
          .limit(1) // 중간 연산: 첫 번째 요소만 필요하다고 명시
          .collect(Collectors.toList()); // 최종 연산: 여기서 모든 중간 연산이 실행됨

      System.out.println("--- 지연 평가 예시 종료 ---");
      System.out.println("결과: " + filteredAndMappedNames);
  }
}
```

**실행 결과:**

```
--- 지연 평가 예시 시작 ---
필터링 중: Alice
매핑 중: Alice
--- 지연 평가 예시 종료 ---
결과: [ALICE]
```

**결과 분석:**

1.  `filter()`와 `map()` 연산 내부에 `System.out.println()`을 넣어 각 연산이 언제 실행되는지 확인했습니다.
2.  `limit(1)` 중간 연산이 있기 때문에, 스트림은 첫 번째 `startsWith("A")` 조건을 만족하는 요소(`"Alice"`)를 찾으면 더 이상 다음 요소(`"Bob"`, `"Charlie"` 등)에 대해 `filter()`나 `map()` 연산을 수행하지 않습니다.
3.  만약 `limit(1)`이 없었다면, 모든 요소에 대해 `filter()`와 `map()` 연산이 실행되었을 것입니다.

이처럼 지연 평가는 필요한 최소한의 작업만 수행하도록 하여, 불필요한 연산을 줄이고 전체적인 성능을 향상시키는 데 크게 기여합니다.
